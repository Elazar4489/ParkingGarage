# ParkingGarage## ניתוח המערכת### הבעיה חניונים לא מסודרים... צריך מערכת שתדע לנהל את המחזור החיים המלא של כל רכב בחניון, מרגע הכניסה ועד ליציאה, תיעוד, חישוב תעריפים וגביית תשלומים.### עבור מי?עבור העירייה שככל הנראה כיום מתמודדת עם חוסר סדר וארגון, חוסר תיעוד והמון כוח אדם מבוזבז.### איך נראית הצלחה?מערכת שמנהלת את מחזור החיים המלא והתקין של רכב בחניון, כולל כניסה, יציאה, חישוב תעריף, גביית תשלום, טיפול נכון בשגיאות ואפשרות להוספה והרחבה היכן שנדרש מבלי לשנות דברים דרמטיים בתוכנית.### מהו תרחיש הכישלון שפוגע באמינות?במקרה של שגיאות, טיפול כללי ושטחי שבעצם מתעלם מהשגיאות - לגודמא זמן יציאה מוקדם מזמן הכניסה - יחושב כמו כל חיוב רגיל, במקרה הזה - תוצאה שלילית (לדוג' -120...) זה טעות! ולא רק טעות אלא פגיעה באמינות ויכול להיות שזה יגרום גם הפסדים לחברה - במקום לחייב את הלקוח הם יזכו אותו.### רכיבים- Car- RegularCar- DisabledVihicle- Motorcycle- ICarLogger- IPreOrder- IPaymentTransaction- ConsoleCarLogger- CreditPayment- CashPayment- RateCalculator- ParkingManager### אחראיות- Car- RegularCar- DisabledVihicle- Motorcycleיודע לבצע את הפעולות הספציפיות שלו - כניסה ויציאה.- ICarLoggerכותב תיעוד של כל פעולה ליעד המבוקש.- ConsoleCarLoggerכותב תיעוד של כל פעולה לטרמינל.- IPreOrderמאפשר הזמנה מראש - מבצע אותה.- IPaymentTransactionיודע לגבות תשלום מהרכב לפי הנתונים שקיבל וע"י כלי שקיבל.- CreditPaymentיודע לגבות תשלום באשראי מהרכב לפי הנתונים שקיבל- CashPaymentיודע לגבות תשלום במזומן מהרכב לפי הנתונים שקיבל- RateCalculatorיודע לקחת Car עם זמן הכניסה והיציאה ולהפוך את זה למחיר לתשלום או לדחות במקרה של שגיאה.- ParkingManagerמתאם את תהליך המערכת - כל כניסה, חישוב, גביה, יציאה, הזמנה מראש, תיעוד ורישום, - לפי הסדר. בלי להחליט איך כל דבר פועל וכו'### תקשורת בין הרכיבים
